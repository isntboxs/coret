## Project Context

This repository is a blank TanStack Start app using React, TypeScript, Vite, and Bun.

Scaffold command used:

```bash
bunx @tanstack/cli@latest create my-tanstack-app --agent
```

Follow-up TanStack Intent commands run:

```bash
bunx @tanstack/intent@latest install
bunx @tanstack/intent@latest list
bunx @tanstack/intent@latest load @tanstack/start-client-core#start-core
bunx @tanstack/intent@latest load @tanstack/react-start#react-start
bunx @tanstack/intent@latest load @tanstack/router-core#router-core
```

The CLI scaffold reported that its internal `bunx --bun @tanstack/intent install --map` step did not complete; the explicit Intent install command above succeeded.

## Stack

- React 19 with TanStack Start and TanStack Router.
- Bun is the package manager and script runner.
- Vite is the build tool.
- TypeScript is configured in bundler mode.
- Vitest is available through the generated `test` script.

## Architecture

- `src/routes/__root.tsx` is the document shell and must keep `HeadContent` in `<head>` and `Scripts` in `<body>`.
- `src/routes/index.tsx` is intentionally blank.
- `src/router.tsx` creates the router from `routeTree.gen`.
- `routeTree.gen.ts` is generated by TanStack tooling and is excluded from editor search/watch by `.vscode/settings.json`.
- `vitest.config.ts` is intentionally separate from `vite.config.ts` so the blank test suite does not boot the Start/Nitro build plugins.
- All application code is isomorphic by default in TanStack Start. Put server-only behavior behind Start server primitives such as `createServerFn` or server-only files.
- Keep `tanstackStart()` in the Vite plugin chain as generated unless TanStack Intent guidance says otherwise for a specific upgrade.
- Do not add runtime app code for CodeRabbit.

## CodeRabbit

CodeRabbit is represented as external repository tooling only. There is no runtime SDK, app code, server function, or workflow for it.

- Configure repository access by installing the CodeRabbit GitHub App or enabling this repository in the CodeRabbit dashboard.
- `.coderabbit.yaml` provides repository review defaults for pull requests.
- No CodeRabbit secrets or environment variables belong in the app.

## Environment

No environment variables are required for the blank app.

If future server-side integrations need secrets, keep them server-side and avoid `VITE_` prefixes unless the value is intentionally public.

## Deployment

The selected deployment adapter is Nitro (agnostic). Build with:

```bash
bun --bun run build
```

Use the generated Nitro output for the target Node-compatible host. Confirm the exact server entry after each Start/Nitro upgrade because output paths can change across versions.

## Known Gotchas

- The first route generation happens during dev/build; do not hand-edit generated route tree output.
- CodeRabbit reviews will not run until the GitHub App/dashboard setup grants repository access.
